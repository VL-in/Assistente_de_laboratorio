Blueprint - Ecossistema de Agentes para P&D Biotech

Objetivo: começar simples (MVP), com base segura e escalável.

Contexto: dados confidenciais, IA local, documentos .docx, .xlsx, .xlsm.

1) Escopo do MVP

  • Ingestão de arquivos por pasta local.
  • Extração de conteúdo e metadados.
  • Indexação vetorial para RAG com embedding local.
  • Chat com resposta baseada em evidência e citação de fonte.
  • Login local simples e trilha de auditoria básica.

Fase 2: SQL em linguagem natural, NER/NEN avançado e multiagentes especializados.

2) Stack Recomendada

Desktop: Tauri + React + TypeScript + Vite Backend local: Python + FastAPI LLM local: LM Studio (API OpenAI-compatible) Embeddings: SentenceTransformers (bge-m3/e5-large); opcional /v1/embeddings no LM Studio Vetorial: Qdrant local Relacional: PostgreSQL (ou SQLite no início) Parsing: python-docx + openpyxl + pandas Orquestração: LangGraph (ou LlamaIndex)

Embeddings no MVP: opção B (recomendada para começar): vetores com SentenceTransformers no backend e LM Studio só para o LLM de resposta. Opção A: modelo de embeddings também no LM Studio via /v1/embeddings.

3) Arquitetura em Camadas

Ingestão

Varre diretórios, detecta novos/alterados e calcula hash.

Processamento

Extrai texto/tabelas, normaliza conteúdo, prepara chunks.

Indexação

Gera embeddings locais e faz upsert no banco vetorial.

Recuperação (RAG)

Busca chunks relevantes por semântica + filtros por metadados.

Agente

Monta contexto e responde com base em evidências do acervo.

UI Desktop

Chat, ingestão, histórico, configurações e auditoria.

4) Fluxo de Dados

  1. Usuário registra pasta local.
  2. Serviço detecta arquivos novos/alterados.
  3. Parser extrai conteúdo e metadados.
  4. Chunking e enriquecimento (opcional NER).
  5. Embeddings gerados localmente.
  6. Upsert no Qdrant + metadados no banco relacional.
  7. Pergunta do chat aciona recuperação top-k.
  8. LLM responde com citações de trechos-fonte.

5) Segurança desde o Início

  • Offline-first: sem tráfego externo por padrão.
  • Criptografia em repouso para bancos/índices locais.
  • Gestão de segredo no cofre do sistema operacional.
  • RBAC simples (admin, pesquisador, revisor).
  • Auditoria de acesso e de consultas realizadas.
  • Guardrails: bloquear instruções maliciosas e exigir fonte na resposta.

6) Estrutura de Projeto (Monorepo)

biotech-assistant/ apps/ desktop/ # Tauri + React api/ # FastAPI services/ ingestion/ indexing/ rag/ security/ packages/ shared-types/ prompts/ infra/ docker/ migrations/ docs/ architecture/ decisions/

7) Roadmap (8 Semanas)

Sprint 1-2: Fundação

  • Monorepo, autenticação básica, banco relacional, logging.

Sprint 3-4: Ingestão

  • Scanner + parser + versionamento por hash.

Sprint 5-6: RAG Local

  • Embedding local, Qdrant, chat com citações.

Sprint 7-8: Segurança e Estabilização

  • Auditoria, RBAC, guardrails, testes integrados e empacotamento desktop.

8) Critérios de Pronto (DoD)

  • Resposta do chat sempre com referência de fonte.
  • Reindexação incremental funcional (somente arquivo alterado).
  • Sem envio de dado para internet no modo offline.
  • Trilha de auditoria por usuário e período.
  • Testes de integração para ingestão + consulta aprovados.
