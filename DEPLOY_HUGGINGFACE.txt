================================================================================
  GUIA DE DEPLOY — ASSISTENTE DE LABORATÓRIO no HUGGING FACE SPACES (gratuito)
================================================================================

Pré-requisitos
--------------
  - Conta gratuita em https://huggingface.co (criar se não tiver)
  - Git instalado localmente
  - Python 3.10+ instalado localmente (para instalar o CLI do HF)
  - Chave de API do OpenRouter (OPENROUTER_API_KEY)
  - Opcionais: LANGFUSE_PUBLIC_KEY / LANGFUSE_SECRET_KEY (observabilidade)

Hardware gratuito que o Space usará
-------------------------------------
  CPU Basic: 2 vCPU | 16 GB RAM | sem GPU | GRATUITO
  Disco ephemeral: 50 GB (apaga ao reiniciar — por isso precisamos do bucket)
  O Space "dorme" após ~30 minutos sem uso e acorda automaticamente.

URL pública gerada automaticamente pelo HF
-------------------------------------------
  https://<seu-usuario>-assistente-de-lab.hf.space

Resultado esperado
-------------------
  - A app Streamlit fica acessível publicamente na URL acima
  - Índice txtai, DuckDB e modelo E5 persistem entre reinicializações
  - Documentos dos projetos ficados no bucket (leitura somente)
  - Secrets (API keys) configurados via painel do HF, nunca no código


================================================================================
  PARTE 1 — CONFIGURAÇÃO INICIAL (fazer uma única vez)
================================================================================

PASSO 1 — Instalar o CLI do Hugging Face
-----------------------------------------
  pip install "huggingface_hub[cli]>=0.23.0"
  hf login
    # Abrirá o browser ou pedirá um token. Gere o token em:
    # https://huggingface.co/settings/tokens  (tipo: Write)


PASSO 2 — Criar o Space no HF
-------------------------------
  1. Acesse https://huggingface.co/new-space
  2. Preencha:
       - Owner: <seu-usuario>
       - Space name: assistente-de-lab
       - License: MIT  (ou outra de sua preferência)
       - SDK: Docker                         ← IMPORTANTE: escolher Docker
       - Visibility: Public (URL pública grátis)
         (Private exige plano pago para acesso externo)
  3. Clique em "Create Space".

  O HF cria um repositório Git vazio em:
    https://huggingface.co/spaces/<seu-usuario>/assistente-de-lab


PASSO 3 — Criar um Storage Bucket (persistência de dados)
----------------------------------------------------------
  O disco do Space é apagado a cada reinicialização.
  O bucket persiste índice txtai, DuckDB e cache de modelos.

  Via web:
    1. Acesse https://huggingface.co/new-bucket
    2. Owner: <seu-usuario>
    3. Nome: assistente-lab-data
    4. Visibility: Private  (seus dados de laboratório)
    5. Clique em "Create Bucket"

  Via CLI (alternativa):
    hf buckets create assistente-lab-data --private

  Depois, associe o bucket ao Space:
    1. Acesse Settings do Space:
       https://huggingface.co/spaces/<seu-usuario>/assistente-de-lab/settings
    2. Em "Attached volumes", clique "Attach a bucket"
    3. Selecione "assistente-lab-data"
    4. Mount path: /data          ← o código já usa /data/* por padrão
    5. Mode: Read & Write
    6. Salve.

  NOTA: O bucket aparece como /data dentro do container em runtime.
  O build-time do Dockerfile NÃO tem acesso ao /data — apenas em runtime.


PASSO 4 — Configurar Secrets no Space
--------------------------------------
  Acesse Settings → Secrets:
    https://huggingface.co/spaces/<seu-usuario>/assistente-de-lab/settings

  Adicione os seguintes secrets (nunca coloque no código):

  OBRIGATÓRIO:
    OPENROUTER_API_KEY        = sk-or-v1-xxxxxxxxxxxx
    OPENAI_API_KEY            = (mesma chave do OpenRouter, para compat LiteLLM)

  OPCIONAIS (observabilidade):
    LANGFUSE_PUBLIC_KEY       = pk-lf-xxxx
    LANGFUSE_SECRET_KEY       = sk-lf-xxxx
    LANGFUSE_BASE_URL         = https://cloud.langfuse.com

  OPCIONAIS (Kaggle — módulo ML):
    KAGGLE_API_TOKEN          = {"username":"...","key":"..."}

  Os secrets ficam disponíveis como variáveis de ambiente em runtime:
    os.environ.get("OPENROUTER_API_KEY")


================================================================================
  PARTE 2 — PREPARAR O CÓDIGO PARA O HF SPACES
================================================================================

PASSO 5 — Criar o Dockerfile para HF Spaces
---------------------------------------------
  O HF Spaces exige:
    - Um Dockerfile na RAIZ do repositório do Space
    - Porta exposta: 7860 (padrão do HF) ou declarada em README.md
    - Usuário com uid=1000 (não root)
    - /data disponível apenas em runtime (não no build)

  Crie o arquivo  docker/hf/Dockerfile  no repositório do projeto:

  ---------- inicio do Dockerfile ----------

  FROM python:3.12-slim-bookworm

  # Dependência de sistema: OpenMP para LightGBM/FLAML
  RUN apt-get update \
      && apt-get install -y --no-install-recommends libgomp1 curl \
      && rm -rf /var/lib/apt/lists/*

  # HF Spaces exige uid=1000
  RUN useradd -m -u 1000 user

  ENV HOME=/home/user \
      PATH=/home/user/.local/bin:$PATH

  WORKDIR $HOME/app

  # Copia apenas requirements primeiro para aproveitar cache de layer
  COPY --chown=user apps/streamlit/requirements-base.txt \
                    apps/streamlit/requirements-security.txt \
                    apps/streamlit/requirements.txt ./

  # Instala dependências (torch CPU — sem GPU no plano gratuito)
  RUN pip install --no-cache-dir --upgrade pip \
      && pip install --no-cache-dir torch --index-url https://download.pytorch.org/whl/cpu \
      && pip install --no-cache-dir -r requirements.txt

  # Modelos spaCy para Presidio (PII)
  RUN python -m spacy download pt_core_news_sm \
      && python -m spacy download en_core_web_sm

  # Cache do sentence-transformers (E5) e ESM-2 apontará para /data em runtime
  # NÃO pré-baixamos aqui: /data não existe no build-time do HF Spaces.
  # O modelo é baixado na primeira execução e fica persistido no bucket /data.
  ENV HF_HOME=/data/huggingface \
      SENTENCE_TRANSFORMERS_HOME=/data/huggingface \
      TRANSFORMERS_CACHE=/data/huggingface

  # Variáveis de runtime do app
  ENV ASSISTENTE_PROJETOS_DIR=/data/projetos \
      ASSISTENTE_TXTAI_DIR=/data/txtai \
      ASSISTENTE_DUCKDB_DIR=/data/duckdb \
      ASSISTENTE_ML_DIR=/data/ml \
      LLM_BASE_URL=https://openrouter.ai/api/v1 \
      LLM_MODEL=openrouter/auto \
      STREAMLIT_SERVER_HEADLESS=true \
      STREAMLIT_BROWSER_GATHER_USAGE_STATS=false

  # Copia o código da aplicação
  COPY --chown=user apps/streamlit/ $HOME/app/

  USER user

  # HF Spaces usa porta 7860 por padrão
  EXPOSE 7860

  HEALTHCHECK --interval=30s --timeout=8s --start-period=60s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://127.0.0.1:7860/_stcore/health', timeout=6).read(64)" || exit 1

  CMD ["streamlit", "run", "app.py", \
       "--server.address=0.0.0.0", \
       "--server.port=7860"]

  ---------- fim do Dockerfile ----------

  ATENÇÃO: O Dockerfile acima deve ficar na raiz do repositório do Space
  (não no subdiretório docker/). Veja o Passo 7 sobre a estrutura do repo.


PASSO 6 — Criar o README.md do Space (metadados obrigatórios)
--------------------------------------------------------------
  O HF usa o bloco YAML no topo do README.md para configurar o Space.
  Crie o arquivo  hf_space/README.md:

  ---------- inicio do README.md ----------

  ---
  title: Assistente de Laboratório
  emoji: 🧬
  colorFrom: blue
  colorTo: green
  sdk: docker
  app_port: 7860
  pinned: false
  ---

  # Assistente de Laboratório

  Assistente RAG multiagente para projetos de P&D em biotecnologia.
  Indexa documentos (.docx, .xlsx, .pdf) e responde perguntas via chat.

  ---------- fim do README.md ----------

PASSO 7 — Estrutura do repositório do Space
--------------------------------------------
  O Space tem seu próprio repositório Git separado do repositório principal
  do projeto. Você vai criar uma pasta local e fazer push para o HF.

  A estrutura esperada na raiz do repositório do Space:

    /                       ← raiz do repo do Space
    ├── README.md           ← metadados do Space (bloco YAML obrigatório)
    ├── Dockerfile          ← Dockerfile único (sem docker-compose)
    └── apps/
        └── streamlit/      ← código copiado do projeto principal
            ├── app.py
            ├── requirements.txt
            ├── requirements-base.txt
            ├── requirements-security.txt
            ├── agents/
            ├── ml/
            ├── olap/
            ├── rag/
            ├── observability/
            ├── evals/
            ├── tests/
            ├── projects_loader.py
            ├── llm_config.py
            └── ...


PASSO 8 — Criar e popular o repositório local do Space
-------------------------------------------------------
  # Clonar o Space recém-criado
  git clone https://huggingface.co/spaces/<usuário>/Assistente-de-Laboratorio

  # Copiar os arquivos do projeto principal
  # (execute a partir da raiz do repositório principal)
  cp -r d:/Vanessa/AI_project/Assistente_de_lab/apps ./apps
  cp -r d:/Vanessa/AI_project/Assistente_de_lab/apps/streamlit ./apps/streamlit

  # Criar o Dockerfile na raiz (conteúdo do Passo 5)
  # Criar o README.md na raiz (conteúdo do Passo 6)

  # Criar .gitignore para não subir arquivos desnecessários
  cat > .gitignore << 'EOF'
  __pycache__/
  *.pyc
  .venv/
  .venv-evals/
  .txtai_data/
  .duckdb_data/
  *.pkl
  Histórico/
  logs/
  EOF


PASSO 9 — Fazer o primeiro push para o HF
------------------------------------------
  cd assistente-de-lab   # raiz do repo do Space

  git add .
  git commit -m "feat: deploy inicial Assistente de Lab"
  git push

  Ao receber o push, o HF inicia automaticamente o build Docker.
  Acompanhe o progresso em:
    https://huggingface.co/spaces/<seu-usuario>/assistente-de-lab

  O build pode levar de 5 a 15 minutos na primeira vez
  (instalação de dependências + spaCy models).
  Builds seguintes são mais rápidos por causa do cache de layers.


PASSO 10 — Subir os documentos dos projetos para o bucket
----------------------------------------------------------
  Os documentos do laboratório ficam em /data/projetos no container.
  Suba-os para o bucket via CLI do HF:

  # Instalar CLI se não instalou ainda
  pip install "huggingface_hub[cli]>=0.23.0"
  hf login

  # Sincronizar pasta local de projetos para o bucket
  hf buckets sync "D:/Caminho/Para/Seus/Projetos" \
      hf://buckets/<seu-usuario>/assistente-lab-data/projetos

  O bucket está montado em /data no container, então os documentos ficam
  acessíveis em /data/projetos — exatamente o que ASSISTENTE_PROJETOS_DIR aponta.

  Para atualizar documentos no futuro, basta rodar o sync novamente.
  Novos arquivos e modificações são detectados automaticamente.


================================================================================
  PARTE 4 — PRIMEIRO USO APÓS O DEPLOY
================================================================================

PASSO 11 — Verificar o Space online
-------------------------------------
  1. Acesse https://huggingface.co/spaces/<seu-usuario>/assistente-de-lab
  2. Aguarde o indicador "Running" no canto superior direito
  3. Na primeira vez que o container inicia, ele faz download do modelo
     sentence-transformers/multilingual-e5-small (~120 MB) para /data/huggingface
     Esse download ocorre apenas uma vez — nas reinicializações seguintes
     o modelo já está no bucket.


PASSO 12 — Indexar os documentos (primeira vez)
------------------------------------------------
  Dentro do app Streamlit:
  1. Acesse a aba "Configuração" ou "RAG"
  2. Clique em "Escanear projetos" para descobrir os arquivos em /data/projetos
  3. Clique em "Construir índice" (ou "Substituir índice" na primeira vez)
  4. Aguarde a indexação — o índice é salvo em /data/txtai no bucket
  5. Nas próximas sessões o índice já existe e é carregado automaticamente

  O índice é persistido no bucket, então indexações futuras são incrementais
  (só processa arquivos novos ou modificados).


PASSO 13 — Usar o chat
------------------------
  1. Na aba "Chat", faça perguntas sobre os documentos indexados
  2. O assistente usa RAG (busca nos documentos) + LLM (OpenRouter)
  3. As respostas citam projeto e arquivo de origem


================================================================================
  PARTE 5 — MANUTENÇÃO E ATUALIZAÇÕES
================================================================================

Atualizar o código
-------------------
  No repositório local do Space:
    git add .
    git commit -m "fix: descrição da mudança"
    git push
  O HF reconstrói o container automaticamente.

Atualizar documentos dos projetos
-----------------------------------
  hf buckets sync "D:/Caminho/Para/Seus/Projetos" \
      hf://buckets/<seu-usuario>/assistente-lab-data/projetos
  Depois, dentro do app, clique em "Atualizar índice" (indexação incremental).

Reiniciar o Space manualmente
-------------------------------
  Settings do Space → "Restart Space"
  Os dados permanecem no bucket; apenas a memória do container é limpa.

Verificar logs
---------------
  Na página do Space, clique em "Logs" para ver stdout/stderr do container.
  Útil para diagnosticar erros de inicialização.

Monitorar uso de memória
--------------------------
  Na página do Space, clique em "Metrics".
  O plano gratuito tem 16 GB RAM — a app usa ~2-3 GB com todos os módulos.

    OPENROUTER_API_KEY         = sk-or-v1-xxxx
    OPENAI_API_KEY             = (igual ao OPENROUTER_API_KEY)
    LANGFUSE_PUBLIC_KEY        = pk-lf-xxxx  (opcional)
    LANGFUSE_SECRET_KEY        = sk-lf-xxxx  (opcional)
    LANGFUSE_BASE_URL          = https://cloud.langfuse.com  (opcional)
    KAGGLE_API_TOKEN           = {"username":...}  (opcional)

  Opcionais de tunagem (adicionar em Settings → Variables):
    RAG_HYBRID_ENABLED         = 1       (padrão; 0 = só semântica)
    RAG_HYBRID_WEIGHT          = 0.4     (peso denso 0–1)
    EMBEDDING_BATCH_SIZE       = 32      (textos por batch no E5)
    LANGFUSE_ENABLED           = 1       (0 = desliga tracing)


================================================================================
  LIMITAÇÕES DO PLANO GRATUITO A TER EM MENTE
================================================================================

  - O Space "dorme" após ~30 min sem uso → cold start de ~30-60s ao acordar
    (o modelo E5 precisa ser carregado do bucket para memória)
  - Sem GPU — inferência do LLM é remota (OpenRouter), não local
  - Módulo ML (FLAML + ESM-2) funciona em CPU; treinamento é lento para
    datasets grandes — recomendado fazer o treino localmente e subir o .pkl
  - Não há suporte a múltiplos containers no mesmo Space (arquitetura
    já adaptada: embedding in-process via sentence-transformers)
  - Porta de saída: apenas 80, 443, 8080 permitidas para chamadas HTTP
    externas — OpenRouter usa 443, OK.
